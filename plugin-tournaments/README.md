# Hidden Tournaments — плагин CTFd

Позволяет выделить группу команд/участников (например, тех, кто играет **вне
зачёта**, но хочет свой рейтинг) в отдельный **бракет** (Bracket — штатная
сущность CTFd 3.7+), скрыть их с общего скорборда и дать им приватный
скорборд, который считается **полностью независимо**: очки за динамические
задания там убывают (decay) по числу решений только внутри этой группы, а не
по всей платформе — и наоборот, решения этой группы никак не двигают цену
задания для основного зачёта.

Плагин не патчит и не подменяет ядро CTFd — работает поверх штатных
`Brackets` и флага `hidden`.

## Что видят участники

Это важный момент, который легко понять неправильно:

- Скрытая команда **продолжает видеть основной скорборд** как обычный
  участник — заходит на `/scoreboard`, видит там основной зачёт. Просто **её
  саму** (и другие скрытые команды) там не показывают — команда не
  заблокирована от просмотра, она просто не попадает в список для *остальных*.
- **Дополнительно** к этому у неё есть свой приватный турнир на `/tournament`
  — со своим независимым рейтингом.

Это два одновременно доступных представления, а не «одно вместо другого».

Обычный (не скрытый) участник на `/tournament` получает редирект на
`/scoreboard` — увидеть чужой скрытый турнир он не может, даже зная его
`bracket_id`.

Админ видит любой турнир, скрытый или нет, через `/admin/tournaments` или
напрямую `/tournament?bracket_id=N`.

## Пошаговая инструкция

### 1. Создать бракет

**Admin → Brackets** (`/admin/brackets`, штатная функция CTFd, плагин её не
трогает). Указать:
- название (например, «Вне зачёта»),
- тип — **обязательно** совпадающий с режимом CTF (`teams` для командного
  режима, `users` для индивидуального).

### 2. Открыть панель плагина

В боковом меню админки — пункт **Tournaments** → `/admin/tournaments`. Список
всех бракетов нужного типа.

### 3. Сделать бракет скрытым

Кнопка **«Make hidden»** напротив бракета. После нажатия:
- у всех, кто уже состоит в этом бракете, сразу выставляется `hidden = true`
  (пропадают из общего скорборда/списка команд/списка решивших задание);
- бракет получает бейдж **«hidden»**.

Кнопка становится **«Make public»** — в любой момент можно вернуть группу в
общий зачёт (снимет `hidden` со всех участников, но из бракета не выкинет —
можно снова скрыть позже, состав сохранится).

Изменения видны на скорборде **сразу**, без задержки.

### 4. Добавить команды в скрытый турнир

В карточке бракета — поле **«Add team by exact name or id»** (или «user», в
зависимости от режима). Точное имя команды или её ID → **Add**. Команда:
- переходит в этот бракет (`bracket_id`),
- автоматически получает `hidden = true`, если бракет уже помечен скрытым.

**Remove** у команды в списке участников — убирает её из бракета, снимает
`hidden`, команда возвращается в обычный зачёт.

### 5. Готово

Игроки из добавленных команд теперь видят у себя приватный рейтинг на
`/tournament` (плюс продолжают видеть основной скорборд как обычно).

## Что нужно понимать про очки

- Стоимость **динамических** заданий на приватном скорборде считается **не**
  от общего числа решений на платформе, а только от числа решений **внутри
  этого конкретного бракета**. Поэтому цифры на общем и на приватном
  скорборде для одного и того же задания могут (и будут) отличаться — это
  ожидаемо, не баг.
- Если скрытых бракетов несколько одновременно — они тоже не влияют друг на
  друга, у каждого своя независимая кривая decay.
- **Статические** задания (без decay) показывают одну и ту же стоимость
  везде — делить тут нечего, сложность на них не распространяется.

## Диагностика

- Бейдж **«visible!»** рядом с участником в `/admin/tournaments` означает,
  что его `hidden` не совпадает с состоянием бракета. При обычной работе
  через кнопки такого быть не должно — сигнал, что кто-то менял флаг `hidden`
  напрямую в БД в обход плагина.
